# Runtime Anchor

Use this one-screen anchor when long chat context, compaction, resume, or repeated continuation risks making the agent forget JARVIS OS behavior.

Do not treat this as a full workflow. It is a reloadable kernel: read it, route to the smallest needed feature, then continue.

## Refresh Triggers

Reread this anchor when:

- context is long, compacted, resumed, or interrupted;
- the user says `continue`, `status`, `what now`, `go`, or says the agent forgot JarvisOS;
- starting or continuing Specialist Mode work;
- before meaningful file edits in a JarvisOS project;
- before final closeout on non-trivial work;
- before saying an artifact, version, report, PDF, build, or public change is ready.

## Runtime Questions

Ask internally:

1. Am I in Core / Light Mode or Specialist Mode?
2. What is the target project, folder, work order, artifact, or memory note?
3. Have I read the smallest relevant current state, project memory, handoff, or task board?
4. Does the task naturally trigger a JarvisOS feature: Work Pack, Reference Router, Skill Dictionary, project skill, Task Lifecycle, Artifact Hygiene, Quality Gate, handoff, or verifier?
5. Am I about to act on ambiguity that should be clarified?
6. Am I about to claim `ready`, `done`, or `verified` without checking the exact artifact or state?
7. Did project state, decisions, blockers, canonical artifacts, or next steps change enough to update memory?

## Minimal Actions

- If the task is small: stay in Core, do the work, verify meaningful output, update memory only if useful.
- If the task is serious or long-running: use Specialist Mode state, roles, work orders, and closeout.
- If context is scattered: create or mentally assemble an Agent Work Pack.
- If a domain skill may apply: scan the Skill Dictionary before loading full skills.
- If a local project skill index exists in Specialist Mode: scan it before choosing the project workflow.
- If output is user-facing or consequential: run the relevant quality/claim/artifact gate.
- If work is unfinished: leave the exact next step and blocker, not vague continuation prose.

## Anti-Drift Rule

Never continue a long JarvisOS thread purely from chat momentum. Refresh this anchor, then route.
