# Learning Log

## Purpose

Capture reusable lessons from real work so the workflow can improve over time without turning every session into a permanent rule.

This is not a diary and not a transcript. It is for operational lessons that may improve future agent behavior.

## Promotion Status

Use one of:

- `candidate`: useful lesson from one session, not yet proven recurring
- `repeated`: seen more than once or across more than one project
- `promoted`: encoded into `AGENTS.md`, the installed skill, templates, a decision entry, or a project note
- `rejected`: considered but intentionally not adopted
- `superseded`: replaced by a newer lesson/rule

## Promotion Rule

A lesson should become a standing workflow rule only when:

- repeated evidence appears,
- or the lesson prevents a high-impact failure,
- or the user explicitly approves the rule.

Do not promote one-off annoyances into permanent rules.
