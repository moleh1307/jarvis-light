# Improvement Proposals

## Purpose

Use this folder for proposed workflow changes that should not be applied immediately.

Create a proposal when an improvement might affect:

- core behavior,
- memory structure,
- templates,
- installed skill behavior,
- project-specific workflow rules,
- irreversible or high-friction changes.

Do not create proposals for obvious small fixes. Apply small low-risk hygiene improvements directly and record them in the relevant session note.

## Proposal Template

```md
# Proposal: Title

Status: proposed

## Context

What triggered this proposal?

## Problem

What recurring friction or risk does this solve?

## Proposed Change

What exactly should change?

## Evidence

What sessions, failures, or repeated patterns support it?

## Risks

What could go wrong if adopted?

## Decision

Approved / rejected / superseded, with link to decision note if relevant.
```
