# Self-Improvement

JARVIS OS is designed to improve, but not impulsively.

The core rule is:

```text
learning candidates are not standing rules
```

## Learning Homes

| File | Purpose |
| --- | --- |
| `system/learning-log.md` | candidate lessons from real work |
| `system/failure-patterns.md` | stable recurring failure modes |
| `system/proposals/` | consequential workflow changes that need review |

## Promotion Lanes

Self-improvement entries can be:

- `candidate`;
- `repeated`;
- `promoted`;
- `rejected`;
- `superseded`.

This prevents one-off observations from becoming permanent behavior.

## Reflexes

Self-improvement reflexes capture tiny reusable signals after meaningful work:

- repeated friction;
- near misses;
- stale state problems;
- generic output patterns;
- missing verification;
- workflow complexity that slowed agents down.

The reflex should be compact. It should route the signal to the right home, not create a new ceremony.

## Learning Pulse

A Learning Pulse is a compact review of recent learning signals:

- what candidates exist;
- what repeated patterns are emerging;
- what should be promoted;
- what should be rejected or archived;
- what to watch next.

It makes JARVIS OS feel more alive without letting it rewrite itself uncontrolled.
