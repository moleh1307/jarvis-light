# Public Artifacts

Public-facing artifacts need framing before execution.

Examples:

- README;
- docs site;
- website;
- landing page;
- public email;
- bio;
- one-pager;
- deployed static site.

## Framing Questions

Before building, infer or ask:

- who will read or judge this?
- what decision should it support?
- what tone builds trust?
- what should it avoid looking like?
- is there a privacy, consent, or deployment boundary?

If the prompt provides a named reference, inspect it before calling the artifact ready.

## Completion Must Distinguish

For deployed work, report:

- local latest files;
- pushed GitHub state;
- production deployment state;
- exact URL if available;
- what remains if production is not updated.

Do not say a public artifact is complete if only local files changed.
