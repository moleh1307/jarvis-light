# Quality Gates

Specialist Mode should not stop at "it works" when the work is public, craft-critical, data-critical, or strategically important.

## Excellence Calibration

Before major work, define:

- reference class;
- quality bar;
- benchmarks;
- rejection tests;
- missing capabilities;
- verification path.

This prevents generic output from becoming the ceiling.

## Stop-the-Line Review

Stop normal execution when continuing would likely produce:

- wrong work;
- generic work;
- stale claims;
- misleading status;
- weak evidence;
- low-quality public artifacts.

Then choose one:

- fix now;
- create corrective work order;
- switch operating mode;
- run claim review;
- escalate only if a real boundary requires it.

## Creative Preproduction

For craft-critical UI, websites, presentations, visuals, and public artifacts:

- classify craft criticality;
- study references and anti-references;
- write an art direction brief;
- compare divergent concepts;
- implement against a concrete target;
- verify with screenshots or rendered output.

## Claim Review

For consequential claims:

1. state the claim;
2. assign a confidence lane;
3. attack it independently;
4. promote, revise, reject, or split it;
5. update downstream work.

Confidence is a workflow object, not a mood.
